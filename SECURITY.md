# Security Policy

## Supported Versions

| Version | Supported |
| ------- | --------- |
| Latest release | Yes |
| Older releases | No |

## Reporting a Vulnerability

If you discover a security vulnerability in Medusa POS, please report it responsibly.

**Do not open a public GitHub issue for security vulnerabilities.**

Instead, please email **security@nari.ge** with:

- A description of the vulnerability
- Steps to reproduce the issue
- Any relevant logs or screenshots
- Your assessment of the severity

We will acknowledge your report within 48 hours and aim to provide a fix or mitigation plan within 7 days for critical issues.

## Scope

This policy applies to the Medusa POS application code in this repository. Vulnerabilities in upstream dependencies (Tauri, Medusa, React, etc.) should be reported to their respective maintainers.

## Recognition

We appreciate the efforts of security researchers who help keep Medusa POS safe. With your permission, we will acknowledge your contribution in the release notes for the fix.
